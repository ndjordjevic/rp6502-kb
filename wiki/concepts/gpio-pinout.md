# GPIO Pinout

**Summary**: GPIO pin assignments for both the RIA Pico and VGA Pico as defined in the firmware source. Not in the web docs — sourced from `src/ria/sys/ria.h`, `cpu.h`, `pix.h`, `com.h`, and `src/vga/sys/pix.h`.

---

## RIA Pico GPIO assignments

| GPIO | Signal | Source constant | Notes |
| --- | --- | --- | --- |
| 0–3 | PIX0–PIX3 | `PIX_PIN_BASE=0, PIX_PIN_COUNT=4` | PIX bus data (4-wire DDR output to VGA Pico) |
| 4 | UART1 TX | `COM_UART_TX_PIN=4` | Serial console (115200 baud) to host |
| 5 | UART1 RX | `COM_UART_RX_PIN=5` | Serial console from host |
| 6 | RIA CS̄ | `RIA_CS_PIN = RIA_PIN_BASE+0` | Chip select from 6502 address decode |
| 7 | RIA RW̄B | `RIA_RWB_PIN = RIA_PIN_BASE+1` | Read/Write from 6502 |
| 8–15 | D0–D7 | `RIA_DATA_PIN_BASE = RIA_PIN_BASE+2` | 6502 data bus (bidirectional) |
| 16–20 | A0–A4 | `RIA_ADDR_PIN_BASE = RIA_PIN_BASE+10` | 6502 address bus (5 bits, selects one of 32 registers) |
| 21 | PHI2 | `CPU_PHI2_PIN=21` | 65C02 clock — **generated by `ria_write` PIO side-set** |
| 22 | IRQB̄ | `CPU_IRQB_PIN=22` | Interrupt request to 6502 (active-low) |
| 26 | RESB̄ | `CPU_RESB_PIN=26` | Reset to 6502 (active-low) — driven only by RIA |

> Only 5 address bits (A0–A4) are decoded. The 6502's full 16-bit address bus is not connected to the RIA; chip-select logic on the board gates access to the RIA's `$FFE0–$FFFF` window.

## VGA Pico GPIO assignments

| GPIO | Signal | Source constant | Notes |
| --- | --- | --- | --- |
| 11 | PHI2 (input) | `PIX_PHI2_PIN=11` | PIX bus clock received from RIA Pico |
| 0–3 (inferred) | PIX0–PIX3 (input) | — | PIX bus data received from RIA Pico |

> The VGA Pico's full GPIO map (VGA DAC outputs, sync signals) is in `src/vga/sys/vga.h` and `scanvideo/scanvideo.pio` — not yet read in detail. The above are confirmed from `src/vga/sys/pix.h`.

## UART console

- **Connector**: The UART1 TX/RX signals come out of the RIA Pico at 115200 baud, 8N1.
- **Purpose**: This is the main monitor/console interface. The VCP (Virtual COM Port) over USB mirrors this same channel.
- `COM_UART = uart1` — not uart0; uart1 is used so that uart0 can be available for other purposes if needed.

---

## PIX bus physical connection

The RIA Pico drives GPIO 0–3 (PIX0–PIX3) at DDR rates synced to PHI2 (GPIO 21). The VGA Pico receives these on its GPIO 0–3, with PHI2 arriving on GPIO 11. Both Picos are on the same PCB; the PIX wires are on-board traces.

See [[pix-bus]] for the frame format and [[pio-architecture]] for which PIO state machines handle these signals.

---

## Notes

- **RIA_PIN_BASE = 6**: The RIA's CS, RWB, and data/address lines all start at GPIO 6. This grouping allows the PIO to sample the bus with compact `in pins N` instructions.
- **PHI2 as side-set**: GPIO 21 is driven by the `ria_write` PIO's side-set, not by a dedicated clock generator peripheral.
- **RESB on GPIO 26**: This is in the ADC pin range on the RP2350, but it's used as a plain digital output here.

## GPIO hardware reference

*Based on [[quadros-rp2040]] — the hardware model that underlies how the RIA firmware configures its pins.*

### GPIO structure

The RP2040 has 56 pins total; 36 are GPIO-capable. Of those, 30 are in the **user bank** (GPIO0–GPIO29) and 6 are in the **QSPI bank** (used internally for external flash — not available for application use). GPIO26–GPIO29 can additionally connect to the ADC.

Each GPIO pin has a **CTRL register** that selects which peripheral drives it (function select) and allows signal override (invert, force-low, force-high).

### Function select

Every GPIO can be assigned to one of up to 9 functions. Key functions used in the RIA:

| Function constant | Peripheral | Notes |
|---|---|---|
| `GPIO_FUNC_SPI` | SPI0 or SPI1 | Not used by current RIA firmware (storage is USB MSC) |
| `GPIO_FUNC_UART` | UART0 or UART1 | Console on GPIO 4–5 |
| `GPIO_FUNC_SIO` | SIO (software control) | Default CPU-controlled GPIO |
| `GPIO_FUNC_PIO0` | PIO block 0 | Bus signals on RIA (GPIO 0–3, 6–21) |
| `GPIO_FUNC_PIO1` | PIO block 1 | Bus signals on RIA; PIX TX |
| `GPIO_FUNC_GPCLK` | Clock output | GPIO 20–25 can output clocks |
| `GPIO_FUNC_NULL` | Disabled | Isolates pin |

All GPIO0–GPIO29 can be assigned to PIO0 (F6) or PIO1 (F7). This is how the RIA firmware gives the PIO state machines direct hardware control of every bus signal.

### PAD configuration

Each GPIO has a **PAD** — the electrical interface between internal logic and the physical pin. Key PAD parameters:

| Parameter | Options | Default | RIA relevance |
|---|---|---|---|
| **Drive strength** | 2mA, 4mA, 8mA, 12mA | 4mA | Data bus (D0–D7) needs higher strength to drive 65C02 at 8 MHz |
| **Slew rate** | SLOW, FAST | SLOW | PHI2 and data outputs benefit from FAST; avoids slow transitions |
| **Schmitt trigger** | enabled/disabled | enabled | Hysteresis on input pins; helps reject noise on 65C02 bus signals |
| **Pull-up / pull-down** | 50k–80k resistors | disabled | Used for signals needing defined idle state |
| **Input buffer** | enabled/disabled | enabled | Can be disabled for analog use or power saving |

**Total GPIO current budget: 50 mA maximum** across all pins. With 8 data bus pins potentially driving simultaneously, drive strength selection matters.

SDK functions (library `hardware_gpio`):
- `gpio_set_slew_rate(gpio, GPIO_SLEW_RATE_FAST)`
- `gpio_set_drive_strength(gpio, GPIO_DRIVE_STRENGTH_8MA)`
- `gpio_set_input_hysteresis_enabled(gpio, true)` — Schmitt trigger
- `gpio_set_pulls(gpio, up, down)` / `gpio_disable_pulls(gpio)`

### Digital I/O registers (SIO)

Three 32-bit registers (one bit per pin, bit 0 = GPIO0):
- **GPIO_OUT**: output state (high/low) when pin is in SIO mode and output enabled
- **GPIO_OE**: output enable (1 = output, 0 = input)  
- **GPIO_IN**: current input state regardless of function or output enable

These registers are part of the SIO (Single-Cycle IO block) and are accessible from both cores simultaneously without bus contention. `gpio_get_all()` reads GPIO_IN in a single cycle.

### GPIO interrupt model

4 event types per pin: `GPIO_IRQ_LEVEL_LOW`, `GPIO_IRQ_LEVEL_HIGH`, `GPIO_IRQ_EDGE_FALL`, `GPIO_IRQ_EDGE_RISE`.

All user-bank GPIO interrupts OR together into a single **IO_IRQ_BANK0** interrupt in each core's NVIC. The handler must check which pin triggered it.

Two callback styles:
- **Normal callback** (one per processor): registered with `gpio_set_irq_callback()` — receives pin number and event mask
- **Raw callback** (multiple allowed): registered with `gpio_add_raw_irq_handler()` — no parameters, zero overhead, maximum performance

> **RIA note**: The RIA does not use GPIO interrupts for bus capture — that is done entirely in PIO hardware. GPIO interrupts are used only for higher-level signals if needed. The PIO IRQ flags (lower 4) are what actually trigger `IO_IRQ_BANK0` from the PIO side — see [[pio-architecture]].
