# 6522 VIA (Versatile Interface Adapter)

**Summary**: The W65C22 Versatile Interface Adapter (VIA) provides two bidirectional 8-bit I/O ports (A and B), two programmable timers (T1, T2), a shift register, and a flexible interrupt system. It is the primary GPIO/timer peripheral for 6502-based systems; the RP6502 includes a W65C22S at a fixed base address.

---

## Overview

The 6522 VIA is a support chip for the 6502 family that provides:
- Two independent 8-bit bidirectional I/O ports (Port A, Port B) with handshaking control lines (CA1/CA2, CB1/CB2)
- Two 16-bit programmable timers (T1 and T2)
- An 8-bit shift register (SR) for serial I/O
- A flexible interrupt system with individual enable bits per source
- Internal address decoding via 4-bit register select (RS3–RS0)

Base address for a VIA is chip-select dependent. On the RP6502, see [[w65c22s]] for the mapped address.

## Internal register map

The VIA is addressed via 4 RS pins (RS3–RS0 = register select, bits 3–0 of address offset).

| Reg # | RS3–RS0 | Read operation | Write operation | Designator |
|-------|---------|---------------|-----------------|------------|
| 0 | 0000 | Input Register B (IRB) | Output Register B (ORB) | ORB/IRB |
| 1 | 0001 | Input Register A (IRA), with handshake | Output Register A (ORA), with handshake | ORA/IRA |
| 2 | 0010 | Data Direction Register B | Data Direction Register B | DDRB |
| 3 | 0011 | Data Direction Register A | Data Direction Register A | DDRA |
| 4 | 0100 | T1 Low-Order Counter | T1 Low-Order Latches | T1C-L |
| 5 | 0101 | T1 High-Order Counter | T1 High-Order Counter (loads both latches → counter) | T1C-H |
| 6 | 0110 | T1 Low-Order Latches (no flag reset) | T1 Low-Order Latches | T1L-L |
| 7 | 0111 | T1 High-Order Latches | T1 High-Order Latches | T1L-H |
| 8 | 1000 | T2 Low-Order Counter (clears T2 flag) | T2 Low-Order Latches | T2C-L |
| 9 | 1001 | T2 High-Order Counter | T2 High-Order Counter (loads low latch → counter, clears T2 flag) | T2C-H |
| 10 | 1010 | Shift Register | Shift Register | SR |
| 11 | 1011 | Auxiliary Control Register | Auxiliary Control Register | ACR |
| 12 | 1100 | Peripheral Control Register | Peripheral Control Register | PCR |
| 13 | 1101 | Interrupt Flag Register | Interrupt Flag Register (write 1 to clear bits) | IFR |
| 14 | 1110 | Interrupt Enable Register (read: bit 7 = 1, bits 0–6 reflect enabled state) | IER (write: bit 7 = 1 to enable, bit 7 = 0 to disable for set bits) | IER |
| 15 | 1111 | Input Register A (no handshake) | Output Register A (no handshake) | ORA/IRA† |

> †Register 15 is the same as Register 1 but without triggering the CA2 handshake strobe.

## I/O ports: Port A and Port B

### Data registers (Reg 0/1)
- **Write** writes to the Output Register (ORB/ORA).
- **Read** returns the input level from the pin (for pins configured as inputs, DDRB/DDRA bit = 0).
- **Input latching** (enabled via ACR): if enabled, the input is latched on each CA1/CB1 active edge, and the read returns the latched value rather than the live pin level.

### Data Direction Registers (Reg 2/3: DDRB, DDRA)
- Each bit controls one pin: **0 = input** (high impedance), **1 = output** (driven by ORB/ORA).
- DDR reset state: all 0 (inputs).

```asm
; Example: Port B = full output, Port A = full input
LDA #$FF
STA DDRB     ; all Port B pins = outputs
LDA #$00
STA DDRA     ; all Port A pins = inputs
```

## Timers T1 and T2

### Timer 1 (T1) — 16-bit countdown with automatic reload

**T1 write sequence**: write low byte to Reg 4 (T1L-L), then write high byte to Reg 5 (T1C-H). Writing Reg 5 simultaneously transfers both bytes into the 16-bit counter and clears the T1 interrupt flag.

**T1 reload** (continuous mode): when the counter reaches 0, it reloads from the 16-bit latch (Regs 6/7) automatically — useful for generating a fixed frequency or periodic interrupt.

**T1 read**: read Reg 4 (T1C-L) to read the low counter byte and simultaneously **clear** the T1 interrupt flag. Read Reg 5 for high byte (does not clear flag). Reading Reg 6/7 reads the latches without affecting the running counter or the flag.

**ACR bits 7:6** control T1 behaviour:

| ACR[7] | ACR[6] | T1 mode |
|--------|--------|---------|
| 0 | 0 | Timed interrupt, one-shot (interrupt each time T1 is loaded; PB7 disabled) |
| 0 | 1 | Continuous interrupts (auto-reload); PB7 disabled |
| 1 | 0 | Timed interrupt, one-shot; PB7 toggles on each interrupt |
| 1 | 1 | Continuous interrupts; PB7 square wave output |

### Timer 2 (T2) — 16-bit countdown, one-shot or pulse count

**T2 write sequence**: write low byte to Reg 8 (T2C-L), then high byte to Reg 9 (T2C-H). Writing Reg 9 loads the counter and clears the T2 interrupt flag.

**ACR bit 5** controls T2 behaviour:
- `0` = one-shot timed interrupt (counts down from loaded value)
- `1` = counts down on each low→high pulse received on PB6 (pulse counting mode)

## Shift Register (SR, Reg 10)

An 8-bit shift register for serial I/O on the CB2 pin, clocked by CB1 (external), T2, or system clock (φ2). **ACR bits 4:2** select the shift mode:

| ACR[4:2] | Mode |
|----------|------|
| 000 | Disabled |
| 001 | Shift in under T2 control |
| 010 | Shift in under φ2 clock |
| 011 | Shift in under external CB1 clock |
| 100 | Shift out free-running at T2 rate |
| 101 | Shift out under T2 control (one-shot) |
| 110 | Shift out under φ2 clock |
| 111 | Shift out under external CB1 clock |

Shift direction: bit 7 is the first out; bit 0 is first in (shifting toward bit 7).

---

## Peripheral Control Register (PCR, Reg 12)

Controls the CA1/CA2 and CB1/CB2 handshake/interrupt lines.

| PCR bits | Function |
|----------|---------|
| [0] | CA1 interrupt active edge: 0 = negative (high→low), 1 = positive (low→high) |
| [3:1] | CA2 mode: 000=neg edge input, 001=neg edge indep interrupt, 010=pos edge input, 011=pos edge indep interrupt, 100=handshake output, 101=pulse output, 110=low output, 111=high output |
| [4] | CB1 interrupt active edge: 0 = negative, 1 = positive |
| [7:5] | CB2 mode: same encoding as CA2 bits 3:1 |

**Handshake output** (CA2/CB2 = `100`): CA2/CB2 goes low when the CPU accesses ORA/ORB, then goes high again when CA1/CB1 signals the peripheral is ready. Used for output-ready signalling.

**Pulse output** (CA2/CB2 = `101`): produces a one-clock-cycle low pulse on every ORA/ORB access. Used for input-acknowledge signalling.

## Auxiliary Control Register (ACR, Reg 11)

| ACR bit(s) | Function |
|------------|---------|
| [0] | Port A input latching: 0 = disabled, 1 = latch on CA1 active edge |
| [1] | Port B input latching: 0 = disabled, 1 = latch on CB1 active edge |
| [4:2] | Shift Register mode (see SR section) |
| [6:5] | T2 mode: see T2 section (bit 5 only used) |
| [7:6] | T1 mode (see T1 section) |

## Interrupt Flag Register (IFR, Reg 13)

Bit 7 is set any time any **enabled** interrupt is pending (regardless of CPU I flag). The ISR should read IFR to identify the source.

| IFR bit | Interrupt source | Cleared by |
|---------|-----------------|-----------|
| 7 | Any enabled interrupt (composite) | All source bits cleared |
| 6 | T1 timeout | Read T1C-L (Reg 4) or write T1C-H (Reg 5) |
| 5 | T2 timeout | Read T2C-L (Reg 8) or write T2C-H (Reg 9) |
| 4 | CB1 active edge | Read or write ORB (Reg 0) |
| 3 | CB2 active edge | Read or write ORB (Reg 0) |
| 2 | Shift Register complete | Read or write SR (Reg 10) |
| 1 | CA1 active edge | Read or write ORA/IRA (Reg 1) |
| 0 | CA2 active edge | Read or write ORA/IRA (Reg 1) |

You can also **write 1 to individual bits** of IFR to clear them without performing the associated read/write. Write 0 to leave those bits unchanged.

---

## Interrupt Enable Register (IER, Reg 14)

Controls which IFR sources drive the `IRQB` line.

- **Read**: always returns bit 7 = 1; bits 0–6 reflect the current enable state of each source.
- **Write with bit 7 = 1**: the set bits in bits 0–6 are **enabled**.
- **Write with bit 7 = 0**: the set bits in bits 0–6 are **disabled**.

```asm
; Enable CA1 (bit 1) and CB1 (bit 4) interrupts:
LDA #%10010010   ; bit 7 = 1 (enable mode), bit 4 (CB1), bit 1 (CA1)
STA VIAIER

; Disable T1 interrupt (bit 6):
LDA #%01000000   ; bit 7 = 0 (disable mode), bit 6 (T1)
STA VIAIER
```

## VIA register address offsets (common mapping)

From Leventhal 1982 Ch. 11B (example using Apple II VIA at base `$C090`):

```asm
VIA     = $C090        ; base address (system-dependent)
VIABDR  = VIA+0        ; Port B Data Register (ORB/IRB)
VIAADR  = VIA+1        ; Port A Data Register with handshake (ORA/IRA)
VIABDD  = VIA+2        ; Port B Data Direction Register
VIAADD  = VIA+3        ; Port A Data Direction Register
VIAACR  = VIA+11       ; Auxiliary Control Register
VIAPCR  = VIA+12       ; Peripheral Control Register
VIAIFR  = VIA+13       ; Interrupt Flag Register
VIAIER  = VIA+14       ; Interrupt Enable Register
```

> **RP6502**: The W65C22S VIA base address is defined in [[w65c22s]]. Offset layout is identical to the standard 6522.

## Typical initialisation sequence

```asm
; Port A = input with latching; Port B = output; CA1/CB1 active low→high
; Enable CA1 (input interrupt) and CB1 (output interrupt)

LDA #%00000000     ; all Port A pins = inputs
STA VIAADD

LDA #%11111111     ; all Port B pins = outputs
STA VIABDD

LDA #%10001010     ; PCR: CA1 pos edge (bit 0=0), CA2 pulse output (bits 3:1=101)
                   ;       CB1 pos edge (bit 4=0), CB2 write strobe (bits 7:5=101)
STA VIAPCR

LDA #%00000001     ; ACR: enable Port A input latching (bit 0=1)
STA VIAACR

LDA #%10010010     ; IER: enable (bit 7=1) CA1 (bit 1) and CB1 (bit 4) interrupts
STA VIAIER
```

## Port A vs Port B differences

| Feature | Port A | Port B |
|---------|--------|--------|
| Read when configured as output | Returns live pin level | Returns ORB value (not pin) |
| Handshake control lines | CA1, CA2 | CB1, CB2 |
| Read triggers handshake | Yes (CA2 pulse on Reg 1 access) | No |
| Timer T1 output | — | PB7 (optional, per ACR[7]) |
| Timer T2 pulse count | — | PB6 (optional, per ACR[5]) |
