# RP2350

**Summary**: The RP2350 is Raspberry Pi's second-generation microcontroller, successor to the RP2040. It powers both the Pi Pico 2 (VGA firmware) and Pi Pico 2 W (RIA-W firmware) used in the RP6502 Picocomputer.

---

## Device family

| Product | Package | Internal Flash | GPIO (user bank) | ADC inputs |
|---|---|---|---|---|
| RP2350A | QFN-60 (7×7 mm) | None | 30 (GPIO0–29) | 4 (GPIO26–29) |
| RP2350B | QFN-80 (10×10 mm) | None | 48 (GPIO0–47) | 8 (GPIO40–47) |
| RP2354A | QFN-60 (7×7 mm) | 2 MB | 30 (GPIO0–29) | 4 (GPIO26–29) |
| RP2354B | QFN-80 (10×10 mm) | 2 MB | 48 (GPIO0–47) | 8 (GPIO40–47) |

The RP6502 uses **RP2350A** (QFN-60, no flash) on both the RIA and VGA modules, booting from external QSPI flash.

## Key features (vs RP2040)

| Feature | RP2040 | RP2350 |
|---|---|---|
| CPU | 2× Cortex-M0+ @ 133 MHz | 2× Cortex-M33 (or Hazard3 RISC-V) @ 150 MHz |
| SRAM | 264 KB (6 banks) | 520 KB (10 banks) |
| PIO blocks | 2 (PIO0, PIO1) | 3 (PIO0, PIO1, PIO2) |
| PIO state machines | 8 | 12 |
| PIO ISA version | v0 | v1 (adds MOV-to/from-RX) |
| SIO TMDS encoder | No | Yes (§3.1.9) — hardware DVI/HDMI encoding |
| HSTX peripheral | No | Yes — high-speed serial transmit for video |
| Low-power oscillator | No | LPOSC (new clock source) |
| GPIO functions | F0–F8 (9 functions) | F0–F11 (12 functions) |
| GPIO count (A/B package) | 30 / — | 30 / 48 |
| Boot security | No | Signed/encrypted boot, OTP keys |
| Max system clock | 133 MHz | 150 MHz (200 MHz with B2+ silicon) |

## Architecture overview

The RP2350 uses an **AHB5 crossbar** bus fabric with 6 upstream managers (core 0 I+D, core 1 I+D, DMA R+W) and up to 17 downstream ports. Simultaneous access to different SRAM banks is fully parallel — key for dual-core + DMA throughput.

Key subsystems:
- **SIO** (Single-Cycle IO): zero-wait-state path for GPIO, spinlocks, FIFOs, doorbells, TMDS encoder, interpolators — see [[dual-core-sio]]
- **PIO**: 3 independent blocks × 4 state machines = 12 total — see [[pio-architecture]]
- **HSTX**: high-speed serial transmit peripheral, likely used for VGA output — see [[hstx]]
- **DMA**: 16 channels with DREQ table, chaining, CRC — see [[dma-controller]]
- **USB**: 1.1 FS/LS host+device, same as RP2040 — see [[usb-controller]]
- **Clocks**: ROSC, XOSC, LPOSC, 2× PLL — see [[rp2040-clocks]]
- **GPIO**: 30/48 user pins, F0–F11 function select — see [[gpio-pinout]]

## SRAM layout

520 KB in 10 independent banks:
- **SRAM0–3**: 4×64 KB, striped on address bits [3:2], base `0x20000000`
- **SRAM4–7**: 4×64 KB, striped on address bits [3:2], base `0x20040000`
- **SRAM8–9**: 2×4 KB, non-striped, base `0x20080000` — useful for stack hoisting

SRAM0–3 in power domain SRAM0; SRAM4–7 in power domain SRAM1; SRAM8–9 in SRAM1.

## Atomic register access

All peripheral register blocks support four access modes via address offset:
- `base + 0x0000` — normal read/write
- `base + 0x1000` — atomic XOR on write
- `base + 0x2000` — atomic bitmask set on write
- `base + 0x3000` — atomic bitmask clear on write

Exceptions: SIO (has its own set/clear/XOR aliases per register), Arm PPB registers, OTP SBPI.

## Relevance to RP6502

1. **VGA firmware** runs on Pi Pico 2 (RP2350A). The TMDS encoder and HSTX peripheral are how DVI/VGA signals are generated. See [[rp6502-vga]] and [[hstx]].
2. **RIA firmware** (Pi Pico 2 / Pi Pico 2 W) uses the 3rd PIO block (PIO2) and RP2350-specific features. The 65C02 bus capture, PIX bus, and PHI2 clock generation all run on PIO state machines.
3. **Doorbell API** (RP2350-only) provides a lightweight cross-core IRQ mechanism used by pico_sync primitives.
4. The RP2350A GPIO bank gives 30 user pins — sufficient for the RIA's bus signals (GPIO6–26) with margin.
