# W65C02S

**Summary**: The Western Design Center CMOS 65C02 — the actual CPU in the Picocomputer. Slot **U1** on [[rp6502-board]]. No substitutions allowed; NMOS 6502s are not supported.

---

## Role on RP6502

- Sole general-purpose CPU. The RP2350 in [[rp6502-ria]] does **not** emulate or offload the 6502; it only provides peripherals, clocks, and the OS.
- Runs at variable **0.1 – 8.0 MHz** (PHI2 generated by [[rp6502-ria]] via PIO).
- PHI2 and RESB are **owned by the RIA** — see [[reset-model]].

## Part number

From [[hardware]]: **W65C02S6TPG-14** — decoded per Ch. 10 of the [[w65c02s-datasheet]]:

| Field | Value | Meaning |
|---|---|---|
| W65C | W65C | Standard WDC product |
| 02S | 02S | W65C02S microprocessor |
| 6T | 6T | 0.6 µm CMOS process, **TSMC** foundry |
| P | P | **Plastic DIP, 40 pins** |
| G | G | RoHS/Green compliant |
| — | (blank) | 0 °C to +70 °C (DIP) |
| -14 | -14 | **14 MHz** speed grade |

The 14 MHz rating at 5 V leaves comfortable headroom for the 8 MHz cap that the RP6502 firmware enforces (`CPU_PHI2_MAX_KHZ 8000`).

## Architecture (from datasheet Ch. 2)

Register file:
- **A** — 8-bit Accumulator
- **X, Y** — 8-bit Index Registers
- **PC** — 16-bit Program Counter
- **S** — 8-bit Stack Pointer (implicit high byte `$01`)
- **P** — 8-bit Processor Status: `N V 1 B D I Z C`
  - N: Negative · V: oVerflow · (bit 5 = 1 unused) · B: BRK flag · D: Decimal · I: IRQ disable · Z: Zero · C: Carry

Functional blocks: Instruction Register + Decode, Timing Control Unit, ALU, Clock Gen/Oscillator, Interrupt Logic, Data Bus Buffer, Input Data Latch.

Note: zero page is **not reserved** — every program starts with a blank 6502.

## Interrupt vectors (Table 3-1)

Fixed, hardware-enforced:

| Address | Vector | Triggered by |
|---|---|---|
| `$FFFA / $FFFB` | NMIB | Non-maskable interrupt (falling edge) |
| `$FFFC / $FFFD` | RESB | Hardware reset |
| `$FFFE / $FFFF` | BRK / IRQB | BRK instruction **or** IRQB (low level) |

Because these live inside the RIA register window (`$FFE0-$FFFF`), the RIA firmware controls what the CPU reads when fetching a vector — see [[ria-registers]].

## Pin signals (datasheet Ch. 3)

Highlights relevant to RP6502:

| Pin | Direction | RP6502 usage |
|---|---|---|
| **A0–A15** | Out | Address bus to VIA/RIA |
| **D0–D7** | In/Out | Data bus |
| **PHI2** | In | **Driven by RIA** via PIO (clock input) |
| **PHI1O / PHI2O** | Out | Unused — RIA generates PHI2 externally |
| **RESB** | In | Held low by RIA during reset windows ([[reset-model]]) |
| **RWB** | Out | Read/Write strobe |
| **IRQB** | In | Typically from [[w65c22s]]; tied high otherwise |
| **NMIB** | In | Edge-triggered; tied high on RP6502 |
| **RDY** | In/Out | Bi-directional; **WAI** pulls low. Latest TSMC parts have no internal pull-up. |
| **SYNC** | Out | High during opcode fetch |
| **VPB** | Out | Low during vector-fetch cycles. Unused by RP6502 firmware. |
| **BE** | In | Tri-states A/D/RWB when low. Kept high on RP6502. |
| **MLB** | Out | Memory Lock for RMW multiprocessor arbitration — ignored |
| **SOB** | In | Sets V flag on negative edge — not used; WDC calls it obsolete |

Packages available: 40-pin **PDIP** (used), 44-pin PLCC, 44-pin QFP.

## Timing (datasheet Ch. 6)

Absolute max VDD: −0.3 V to +7.0 V. Storage temp: −55 °C to +150 °C.

Max PHI2 frequency by supply voltage:

| VDD | F max |
|---|---|
| 5.0 V ±5% | **14 MHz** |
| 3.3 V ±10% | **8 MHz** |
| 3.0 V ±5% | 8 MHz |
| 2.5 V ±5% | 4 MHz |
| 1.8 V ±5% | 2 MHz |

RP6502 runs the CPU at 5 V / 8 MHz max — well inside the envelope. Core supply current is ~150 µA/MHz; standby (PHI2 frozen) is ~1 µA. Fully static: **PHI2 can be stopped indefinitely** in either state without losing register contents.

## WDC enhancements over NMOS 6502 (Ch. 7)

The W65C02S is not just a CMOS die shrink — it fixes bugs and adds features. Summary (full details in [[w65c02s-datasheet]]):

- **New instructions**: BBR/BBS, BRA, PHX/PHY/PLX/PLY, RMB/SMB, STP, STZ, TRB/TSB, WAI.
- **New addressing modes**: `(zp)`, `(a,x)` for JMP.
- Invalid opcodes are **deterministic NOPs** (NMOS had undefined behavior).
- Decimal flag D is **cleared on reset/IRQ**.
- Flags **valid after decimal ADC/SBC** (costs +1 cycle).
- **JMP (XXFF)** page-wrap bug **fixed**.
- **RMW** sequence is now read-read-write (was read-write-write).

See [[65c02-instruction-set]] and [[65c02-addressing-modes]] for the full set.

## Hardware substitution

From [[hardware]]:
- Must be a genuine WDC **W65C02S** (part `W65C02S6TPG-14`).
- **No NMOS**: don't try a 6502 without the "C" in the part number.
- Old CMOS variants "may work" but there are no plans to support out-of-production parts.

## Open questions

- Exact speed ceiling with different glue-logic CMOS families (AC vs. HC) — the source only says "two of three must be AC for 8 MHz".
- Whether the W65C02S's WAI and STP instructions interact usefully with the RIA's `RIA_SPIN` polling mechanism (see [[rp6502-abi]]). The datasheet confirms WAI pulls RDY low and releases on any interrupt — could replace the busy-polling JSR stub, but would require an IRQB source on each RIA completion.
