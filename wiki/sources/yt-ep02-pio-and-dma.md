# Ep2 — 6502 reads Raspberry Pi Pico @ 8 MHz - PIO and DMA

**Summary**: The dual-Pico design is finalized and the 6502 read path is implemented — PIO reads the address bus and builds a register address; chained DMA pushes the register value back onto the data bus. The system achieves 8 MHz by doubling the Pi Pico clock.

**Video**: [https://www.youtube.com/watch?v=GOEI2OpMncY](https://www.youtube.com/watch?v=GOEI2OpMncY)

---

## Key topics

- **Pivot to dual-Pico design** — author decided to "go all in" on two Pi Picos: "less parts, more RAM, cheaper, faster, better video and sound." This locked in the [[rp6502-ria]] + [[rp6502-vga]] architecture.
- **Memory map introduction** — no ROM; 64 K all-RAM. Top page (`$FF00–$FFFF`) reserved for I/O. Pi Pico RIA occupies `$FFE0–$FFFF` (32 bytes), including the 6502 reset/interrupt vectors. 10 of the 26 usable I/O bytes are enough for a bootloader.
- **PHI2 and RESB** — PHI2 generated by GPIO; RESB must be held low for ≥ 2 PHI2 cycles.
- **PIO read program** — `IN` reads address bus; `PULL`/`MOV` constructs a 32-bit register address from a pre-loaded base address + lower 5 GPIO bits. Data moves through TX/RX FIFOs.
- **Chained DMA read loop (8 steps)**: (1) PIO builds register address, (2) PIO pushes to FIFO, (3) address DMA reads FIFO, (4) address DMA writes result as source pointer into data DMA's control block, (5) data DMA reads register value, (6) data DMA pushes to FIFO, (7) PIO reads FIFO, (8) PIO drives data bus. Both CPUs can halt and the loop keeps running.
- **Achieving 8 MHz** — initial access time: adequate for 1 MHz; 4 MHz required a WAIT instruction to sync PIO to PHI2 falling edge (2-cycle empirical delay); 8 MHz required doubling the Pi Pico system clock.
- **Logic analyzer debugging** — used a minimal 3-byte JMP loop as a glitch detector; any address bus error causes the CPU to leave the loop.
- **PIO vs bitbang rationale** — "bitbanging uses 100% of a CPU; DMA+PIO use a fraction of those resources and none of the CPU."
